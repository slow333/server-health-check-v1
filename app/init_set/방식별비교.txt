paramiko, fabric, pexpect, asyncssh는 모두 Python에서 SSH 통신을 처리하는 라이브러리이지만, 설계 목적과 추상화 수준에서 명확한 차이가 있습니다.
주요 차이점은 다음과 같습니다.
라이브러리 	 추상화 수준	        주요 용도	             비동기 지원	             기반 기술/특징
Paramiko   낮음 (Low-level)	SSHv2 프로토콜 구현, 원격 셸 및 파일 전송(SFTP/SCP) 제어	아니요 (동기)	 
            SSH 프로토콜 자체를 Python으로 구현한 순수 Python 라이브러리
Fabric	  높음 (High-level)	애플리케이션 배포, 시스템 관리 작업 자동화, 다중 서버 명령 실행	  아니요 (동기)	
             Paramiko를 기반으로 구축되어 더 간편한 고수준 API 제공
Pexpect   다름 (Expect-style)	자동화된 대화형 프롬프트 처리 (예/아니오 응답, 비밀번호 입력 등)	아니요 (동기)	
             SSH 세션을 직접 제어하기보다, 터미널 프로세스를 스폰하고 입출력을 기대/응답하는 방식
AsyncSSH  중간 (Mid-level)	고성능 동시 SSH 연결, 비동기 프로그래밍	예 (비동기)	
             Python의 asyncio 프레임워크를 활용하여 비동기 SSH 클라이언트/서버 구현

각 라이브러리의 상세 특징은 다음과 같습니다.
Paramiko:
    특징: 순수 Python으로 작성된 저수준(low-level) SSHv2 프로토콜 라이브러리입니다. SSH 클라이언트와 서버를 모두 구현할 수 있으며, 연결, 인증, 명령 실행, SFTP 파일 전송 등 SSH의 모든 기능을 직접 제어해야 합니다.
    사용 시점: SSH 데몬 구현 등 세밀한 SSH 프로토콜 제어가 필요하거나 Fabric보다 더 깊은 수준의 코딩이 필요할 때 적합합니다.
Fabric:
    특징: Paramiko를 기반으로 구축되었으며, 원격 시스템 관리 및 애플리케이션 배포 작업을 간소화하는 고수준(high-level) 명령줄 도구 및 라이브러리입니다. 여러 서버에서 동일한 작업을 쉽게 병렬/순차적으로 실행할 수 있습니다.
    사용 시점: 인프라 관리, 배포 스크립트 작성 등 반복적인 시스템 관리 작업 자동화에 최적입니다.
Pexpect:
    특징: 다른 라이브러리들과는 달리 SSH 프로토콜을 직접 구현하지 않습니다. 대신 로컬 터미널에서 실행되는 SSH 프로세스를 제어하며, 예상되는 출력(expect)에 따라 자동으로 응답(send)을 보냅니다. 사용자 상호작용이 필요한 비밀번호 프롬프트 등에 유용합니다.
    사용 시점: Telnet 세션 제어, 사용자 입력이 필요한 CLI 명령 자동화 등 대화형 세션 자동화에 주로 사용됩니다.
AsyncSSH:
    특징: Python의 asyncio 라이브러리를 활용하여 비동기 SSH 연결을 지원합니다. 여러 SSH 연결을 동시에 효율적으로 관리할 수 있어 성능 면에서 유리합니다. Paramiko와 유사한 기능을 제공하지만 비동기 방식으로 작동합니다.
    사용 시점: 수백, 수천 개의 장치에 동시에 연결하여 데이터를 수집하는 등 고성능 및 동시성이 요구되는 네트워크 프로그래밍에 적합합니다
